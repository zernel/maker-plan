# 🔌 实体计算 (Physical Computing)

> **一句话定义**：通过传感器、微控制器和代码创建能与物理世界互动的交互系统。

---

## 📖 清晰定义

实体计算（Physical Computing）是**连接数字世界和物理世界的桥梁**：

- **核心理念**：让计算机"感知"物理世界并做出响应
- **关键组件**：传感器（输入）+ 微控制器（处理）+ 执行器（输出）
- **学习价值**：将抽象代码转化为有形、可触摸的结果
- **应用领域**：智能家居、可穿戴设备、互动艺术、机器人

> "Physical computing is about creating interactive systems that can sense and respond to the world around them."
> — Dan O'Sullivan

---

## 🎯 为什么实体计算很重要？

### 1. 🎯 具体化结果 (Concrete Results)
- 代码执行结果可视化、可触摸
- 从"屏幕上的数字"到"闪亮的 LED"
- 增强学习动机和成就感

### 2. 🧠 多感官学习 (Multi-sensory Learning)
- 视觉：LED、屏幕显示
- 听觉：蜂鸣器、音乐
- 触觉：按钮、传感器
- 动觉：搭建、连接

### 3. 🌍 真实世界关联 (Real-world Relevance)
- 解决真实问题：温度监测、安防系统
- 理解物联网（IoT）基础
- 培养工程思维

### 4. ⚡ 即时反馈 (Immediate Feedback)
- 代码修改 → 立即看到物理效果
- 快速迭代和调试
- 降低学习挫败感

## 🔧 常见平台

### 🐝 Bee-Bot / Blue-Bot
**适合年龄**：4-6 岁

**特点**：
- 无屏幕、按钮编程
- 即时物理反馈
- 低门槛、高趣味性

**学习内容**：
- 顺序概念
- 方向和距离
- 空间推理

**项目示例**：
- 迷宫探险
- 地图导航
- 舞蹈编程

### 🎨 micro:bit
**适合年龄**：7-12 岁

**特点**：
- 图形化编程（MakeCode）+ 文本编程（Python）
- 内置传感器（加速度计、光线、温度）
- 丰富的扩展模块

**学习内容**：
- 传感器应用
- 条件逻辑
- [[计算思维]]

**项目示例**：
- 温度显示器
- 计步器
- 电子宠物

### 🤖 Arduino
**适合年龄**：10 岁+

**特点**：
- 开源硬件平台
- C/C++ 编程
- 丰富的传感器和扩展

**学习内容**：
- 电子基础
- 编程逻辑
- 系统设计

**项目示例**：
- 智能浇花系统
- 超声波测距仪
- 交通灯模拟器

### 🍓 Raspberry Pi
**适合年龄**：12 岁+

**特点**：
- 完整的微型计算机
- 支持多种编程语言（Python、Scratch）
- 可运行操作系统

**学习内容**：
- 计算机原理
- 网络编程
- 人工智能基础

**项目示例**：
- 家庭媒体中心
- 网络服务器
- 人脸识别系统

## 👶 年龄适宜性项目

### 🌱 4-6 岁：感官探索

**项目：魔法盒子**
- 材料：纸盒、LED、按钮、电池
- 目标：按下按钮，LED 亮起
- 学习：因果关系、电路基础

**项目：声音花园**
- 材料：Makey Makey、水果、Scratch
- 目标：触摸水果发出声音
- 学习：导体概念、触发机制

### 🌿 7-9 岁：传感器入门

**项目：环境监测站**
- 材料：micro:bit、温度传感器、光线传感器
- 目标：显示教室环境数据
- 学习：传感器原理、数据读取

**项目：智能交通灯**
- 材料：Arduino、LED、按钮
- 目标：模拟交通灯控制系统
- 学习：状态机、时序控制

### 🌳 10-12 岁：系统设计

**项目：自动浇花系统**
- 材料：micro:bit、土壤湿度传感器、水泵
- 目标：根据土壤湿度自动浇水
- 学习：传感器融合、控制逻辑
- 融合：[[机器人教育]]、[[STEAM教育]]

**项目：入侵检测器**
- 材料：Arduino、PIR 传感器、蜂鸣器
- 目标：检测移动并报警
- 学习：传感器应用、条件逻辑

### 🌲 12 岁+：复杂系统

**项目：气象站**
- 材料：Raspberry Pi、多种传感器、网络模块
- 目标：收集并上传天气数据
- 学习：数据采集、网络通信、数据可视化

**项目：机器人避障车**
- 材料：Arduino、超声波传感器、电机驱动
- 目标：自动避开障碍物
- 学习：传感器融合、控制算法
- 参赛：[[01-White-List-Competitions/02-WRC-Robot-Design|WRC 世界机器人大赛]]

## 🛠️ 入门建议

### 第一步：选择平台

| 年龄 | 推荐平台 | 理由 |
|------|----------|------|
| 4-6 岁 | Bee-Bot、Makey Makey | 简单、安全、有趣 |
| 7-9 岁 | micro:bit | 图形化编程、丰富传感器 |
| 10-12 岁 | Arduino | 开源、社区支持、扩展性强 |
| 12 岁+ | Raspberry Pi | 功能强大、可学习计算机原理 |

### 第二步：从简单项目开始

**原则**：
- 单一传感器 + 单一输出
- 明确的目标
- 可视化结果

**示例**：
- micro:bit + LED = 温度显示器
- Arduino + 按钮 + LED = 简单开关

### 第三步：逐步增加复杂度

**进阶路径**：
1. 单一传感器 + 单一输出
2. 多个传感器 + 单一输出
3. 传感器 + 条件逻辑 + 输出
4. 传感器 + 网络 + 云端

## 💡 实践案例

### 案例 1：校园噪音监测（8-10 岁）
**项目目标**：监测教室噪音水平，超过阈值时提醒

**技术方案**：
- 硬件：micro:bit + 声音传感器 + LED
- 编程：读取声音数据 → 判断阈值 → 控制 LED
- 学习：传感器应用、条件逻辑、[[计算思维]]

### 案例 2：智能垃圾桶（11-13 岁）
**项目目标**：检测垃圾类型并自动分类提示

**技术方案**：
- 硬件：Arduino + 颜色传感器 + 舵机 + LCD
- 编程：识别颜色 → 判断类型 → 显示提示
- 学习：传感器融合、系统设计、[[STEAM教育]]

### 案例 3：社区健康监测（14+ 岁）
**项目目标**：监测社区空气质量并预警

**技术方案**：
- 硬件：Raspberry Pi + PM2.5 传感器 + 温湿度传感器 + 网络模块
- 编程：数据采集 → 云端上传 → 可视化展示
- 学习：物联网、数据分析、[[创客教育]]
