# 💡 设计思维 (Design Thinking)

> **一句话定义**：以人为本的创新方法论，通过共情、定义、构思、原型、测试五个步骤解决问题。

---

## 🧒 给孩子的解释

想象你的好朋友不开心，你想帮他。你会：

1. **先了解他为什么不开心**（共情）
2. **搞清楚到底是什么问题**（定义）
3. **想很多很多办法**（构思）
4. **选一个办法试试看**（原型）
5. **看看有没有效果**（测试）

这就是设计思维——**先理解别人，再解决问题**。

---

## 🧠 五步法详解

### 步骤流程

```
共情 → 定义 → 构思 → 原型 → 测试
 ↓      ↓      ↓      ↓      ↓
理解   聚焦   发散   快速   验证
用户   问题   想法   实现   迭代
```

### 各步骤说明

| 步骤 | 核心问题 | 儿童版问题 | 活动示例 |
|------|----------|------------|----------|
| **共情** | 用户需要什么？ | "你的朋友需要什么？" | 访谈、观察、体验 |
| **定义** | 要解决什么问题？ | "到底是什么让他不开心？" | 问题陈述、需求分析 |
| **构思** | 有哪些可能的解决方案？ | "你能想出10个办法吗？" | 头脑风暴、思维导图 |
| **原型** | 快速做一个试试 | "先做一个简单的看看" | 积木搭建、草图绘制 |
| **测试** | 效果怎么样？ | "你的朋友喜欢吗？" | 用户测试、收集反馈 |

## 🎮 儿童版设计思维活动

### 活动1：为朋友设计礼物（4-6岁）

**目标**：体验共情和构思

**步骤**：
1. 问孩子："你的好朋友最喜欢什么？"
2. 想3个可能的礼物
3. 选一个，用积木做出来
4. 送给朋友，看反应

**学习点**：
- 理解别人的需求
- 发散思维
- 快速原型

### 活动2：设计更好的书包（7-9岁）

**目标**：完整体验五步法

**步骤**：
1. **共情**：采访同学，书包有什么问题？
2. **定义**：总结问题，如"书包太重"、"找不到东西"
3. **构思**：想10个改进方案
4. **原型**：画设计图或用纸板做模型
5. **测试**：让同学试用，收集反馈

**学习点**：
- 用户研究
- 问题定义
- 创意发散
- 快速原型
- 迭代优化

### 活动3：解决校园问题（10-12岁）

**目标**：应用设计思维解决真实问题

**步骤**：
1. **共情**：观察校园，发现问题
2. **定义**：选择一个要解决的问题
3. **构思**：团队头脑风暴
4. **原型**：用Arduino/Micro:bit做原型
5. **测试**：在校园测试，收集数据

**学习点**：
- 社会责任感
- 团队协作
- 技术应用
- 项目管理

## 📋 设计思维工具箱

### 共情工具

**用户访谈模板**：
```
1. 你最喜欢____的什么？
2. 你最讨厌____的什么？
3. 如果可以改变一件事，你想改变什么？
4. 你希望____能做什么？
```

**观察记录表**：
```
观察对象：____
观察时间：____
观察地点：____
我看到：____
我听到：____
我感受到：____
```

### 构思工具

**头脑风暴规则**：
1. 不批评任何想法
2. 数量比质量重要
3. 鼓励疯狂的想法
4. 在别人的想法上发展

**思维导图模板**：
```
          [核心问题]
         /    |    \
    [想法1] [想法2] [想法3]
      |       |       |
   [细节]  [细节]  [细节]
```

### 原型工具

**快速原型方法**：
- 纸板模型
- 积木搭建
- 草图绘制
- 角色扮演
- 故事板

### 测试工具

**用户测试问题**：
```
1. 你觉得这个____怎么样？
2. 哪里好？哪里不好？
3. 如果可以改一件事，你想改什么？
4. 你会用这个____吗？
```

## 📝 家长指导

### 如何引导孩子使用设计思维？

1. **从共情开始**：让孩子理解别人的需求
2. **鼓励发散**：想出尽可能多的解决方案
3. **快速原型**：不要追求完美，先做出来
4. **重视反馈**：认真听取用户的意见
5. **迭代优化**：根据反馈不断改进

### 常见问题

**Q：孩子太小，不会共情怎么办？**
A：从简单的问题开始。"你的好朋友喜欢什么颜色？"逐步引导。

**Q：孩子想不出10个想法怎么办？**
A：降低要求，先想3个。或者用"如果...会怎样？"来引导。

**Q：原型太粗糙怎么办？**
A：没关系！重点是快速验证想法，不是做精品。
