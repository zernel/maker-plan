## 🧠 核心原理简述（给家长）

**传感器融合**，简单说就是"让多个传感器一起合作，得出一个更靠谱的结论"。

为什么需要融合？单个传感器有局限：
- **超声波传感器**测距离很准，但遇到玻璃或软布会"失灵"
- **红外传感器**检测物体很快，但受环境光线影响大
- **温度传感器**读数稳定，但反应慢

把它们的数据组合起来，就像"三个臭皮匠，赛过诸葛亮"。

常用的融合方法：
- **加权平均**：给不同传感器分配"信任权重"，取加权平均值。超声波测得10cm，红外测得12cm，如果更信任超声波，就给它更高权重
- **投票机制**：多个传感器投票决定结果。3个传感器中2个说"有障碍物"，就判定有障碍物
- **卡尔曼滤波**（简介）：一种数学方法，能根据传感器的历史数据预测下一次读数，过滤掉噪声。10-12岁的孩子理解原理即可，不需要手写公式

竞赛中为什么重要？机器人在复杂环境中比赛，单一传感器经常误判。融合多个传感器的数据，机器人就能做出更稳定、更准确的决策，这是拿高分的关键。

## 🔬 趣味实验清单

### 实验1: 🚗 超声波+红外双传感器避障

**零件清单**:
- ESP32 开发板 × 1
- HC-SR04 超声波传感器 × 1（参考 [[03-Hardware-Sensors/超声波传感器|超声波传感器]]）
- 红外避障传感器 × 1（参考 [[03-Hardware-Sensors/红外避障传感器|红外避障传感器]]）
- 面包板 × 1
- 杜邦线若干
- USB 数据线 × 1

**玩法描述**:
1. 分别读取两个传感器的数据，观察各自的特点：
```python
from machine import Pin, time_pulse_us
import time

# 超声波传感器
trig = Pin(14, Pin.OUT)
echo = Pin(12, Pin.IN)

# 红外避障传感器
ir = Pin(27, Pin.IN)

def get_distance():
    trig.value(0)
    time.sleep_us(2)
    trig.value(1)
    time.sleep_us(10)
    trig.value(0)
    duration = time_pulse_us(echo, 1, 30000)
    distance = duration * 0.0343 / 2
    return distance

# 单独读取
dist = get_distance()
ir_val = ir.value()  # 0=有障碍, 1=无障碍
print(f"超声波距离: {dist:.1f}cm, 红外: {'有障碍' if ir_val == 0 else '无障碍'}")
```

2. 实现融合判断逻辑：
```python
def detect_obstacle():
    """双传感器融合：投票机制"""
    dist = get_distance()
    ir_blocked = (ir.value() == 0)
    
    ultrasonic_blocked = dist < 20  # 超声波判断20cm内有障碍
    
    # 投票机制：两个都说有障碍才算有
    if ultrasonic_blocked and ir_blocked:
        return "确认有障碍物，立即转向！"
    elif ultrasonic_blocked or ir_blocked:
        return "可能有障碍，减速观察"
    else:
        return "道路畅通，继续前进"

while True:
    result = detect_obstacle()
    print(result)
    time.sleep(0.3)
```

**观察与思考**:
- 超声波和红外传感器各自在什么情况下会"误判"？
- 投票机制和"任一触发就报警"相比，各有什么优缺点？
- 如果再加一个传感器，投票该怎么设计？

### 实验2: 🌡️ 温湿度+光线环境监测融合

**零件清单**:
- ESP32 开发板 × 1
- DHT11 温湿度传感器 × 1（参考 [[03-Hardware-Sensors/温湿度传感器|温湿度传感器]]）
- 光敏电阻模块 × 1（参考 [[03-Hardware-Sensors/光敏电阻|光敏电阻]]）
- 面包板 × 1
- 杜邦线若干
- OLED 显示屏（可选，参考 [[03-Hardware-Sensors/OLED显示屏|OLED显示屏]]）

**玩法描述**:
1. 读取三种环境数据并计算"舒适度指数"：
```python
from machine import Pin, ADC
import dht
import time

# 温湿度传感器
dht_sensor = dht.DHT11(Pin(4))

# 光敏电阻（模拟输入）
light_adc = ADC(Pin(34))
light_adc.atten(ADC.ATTN_11DB)  # 0-3.3V范围

def read_environment():
    dht_sensor.measure()
    temp = dht_sensor.temperature()
    humi = dht_sensor.humidity()
    light = light_adc.read()  # 0-4095，越大越亮
    return temp, humi, light

def comfort_index(temp, humi, light):
    """加权平均计算舒适度指数 (0-100)"""
    # 温度评分：25度得满分，偏离越多分越低
    temp_score = max(0, 100 - abs(temp - 25) * 5)
    # 湿度评分：50%得满分
    humi_score = max(0, 100 - abs(humi - 50) * 2)
    # 光线评分：中等亮度得满分
    light_pct = light / 4095 * 100
    light_score = max(0, 100 - abs(light_pct - 50) * 1.5)
    
    # 加权平均：温度权重40%，湿度30%，光线30%
    total = temp_score * 0.4 + humi_score * 0.3 + light_score * 0.3
    return round(total, 1)

while True:
    t, h, l = read_environment()
    ci = comfort_index(t, h, l)
    print(f"温度:{t}℃ 湿度:{h}% 光线:{l} 舒适度:{ci}")
    time.sleep(2)
```

2. 尝试调整权重值，观察舒适度指数的变化。

**观察与思考**:
- 三个传感器的读数变化速度一样吗？谁最快，谁最慢？
- 权重不同，舒适度指数会怎么变？哪个传感器最重要？
- 如果要给"植物生长环境"设计舒适度指数，权重该怎么调？

### 实验3: 🤖 多传感器数据日志与可视化

**零件清单**:
- ESP32 开发板 × 1
- HC-SR04 超声波传感器 × 1
- DHT11 温湿度传感器 × 1
- 光敏电阻模块 × 1
- 面包板 × 1
- 杜邦线若干
- 电脑（用于数据可视化）

**玩法描述**:
1. 同时采集多个传感器数据，保存为日志：
```python
from machine import Pin, ADC, time_pulse_us
import dht
import time

# 初始化传感器
trig = Pin(14, Pin.OUT)
echo = Pin(12, Pin.IN)
dht_sensor = dht.DHT11(Pin(4))
light_adc = ADC(Pin(34))
light_adc.atten(ADC.ATTN_11DB)

def get_distance():
    trig.value(0)
    time.sleep_us(2)
    trig.value(1)
    time.sleep_us(10)
    trig.value(0)
    duration = time_pulse_us(echo, 1, 30000)
    return duration * 0.0343 / 2

def collect_data():
    """采集所有传感器数据"""
    dht_sensor.measure()
    return {
        "distance": round(get_distance(), 1),
        "temp": dht_sensor.temperature(),
        "humi": dht_sensor.humidity(),
        "light": light_adc.read()
    }

def simple_kalman(estimated, measured, confidence=0.8):
    """简化版卡尔曼滤波（加权平均思想）
    confidence: 对预测值的信任程度 (0-1)
    """
    return estimated * confidence + measured * (1 - confidence)

# 数据采集与滤波
history = {"distance": 0, "temp": 0, "humi": 0, "light": 0}

for i in range(20):
    data = collect_data()
    
    # 对每个传感器应用简化滤波
    for key in data:
        history[key] = simple_kalman(history[key], data[key])
    
    print(f"[{i+1}] 原始: 距离{data['distance']}cm "
          f"滤波后: 距离{history['distance']:.1f}cm")
    time.sleep(1)
```

2. 用电脑端 Python（Thonny 或 Mu）读取串口数据，画出折线图：
```python
# 电脑端代码（需要 pyserial 和 matplotlib）
import serial
import matplotlib.pyplot as plt

ser = serial.Serial('/dev/ttyUSB0', 115200)  # Windows用COM口
distances = []

for _ in range(50):
    line = ser.readline().decode().strip()
    if "距离" in line:
        val = float(line.split("距离")[1].split("cm")[0])
        distances.append(val)

plt.plot(distances, label="距离")
plt.xlabel("采样次数")
plt.ylabel("距离 (cm)")
plt.title("传感器数据可视化")
plt.legend()
plt.show()
```

**观察与思考**:
- 滤波前后的数据曲线有什么不同？
- 哪个传感器的数据波动最大？为什么？
- 如果滤波的 confidence 值设成 0.5 和 0.95，效果有什么区别？

## 📝 家长小贴士

> 💡 **关键点**: 传感器融合的核心思想是"不把鸡蛋放在一个篮子里"。让孩子理解：多个不完美的信息源组合起来，比一个信息源更可靠。这个思维在生活中也适用。

> ⚠️ **常见误区**: 不要一上来就教卡尔曼滤波的数学公式。先让孩子用"投票"和"加权平均"建立直觉，理解"为什么要融合"比"怎么融合"更重要。

> 🔧 **动手建议**: 准备不同材质的障碍物（纸板、布料、玻璃杯），让孩子分别用超声波和红外测试，亲眼看到单个传感器的"盲区"，这样他们对融合的需求就有切身体会。
