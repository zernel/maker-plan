## 🧠 核心原理简述（给家长）

**算法** 是解决问题的步骤和方法。就像做菜有菜谱，算法就是计算机的"菜谱"。

为什么学算法：
- **效率差异**：同样找一本书，遍历整个图书馆 vs 查目录索引，速度差百倍
- **竞赛核心**：NOI 信息学奥赛 70% 的题目考算法，不会算法就无法解题
- **思维训练**：算法培养分解问题、抽象建模、优化方案的计算思维
- **升学加分**：掌握算法是科技特长生的重要考核标准

三大基础算法：
- **排序 (Sorting)**：把杂乱的数据按规则排列，如成绩排名
- **搜索 (Searching)**：从大量数据中找到目标，如查字典
- **递归 (Recursion)**：大问题拆成小问题，自己调用自己

## 🔬 趣味实验清单

### 实验1: 🃏 冒泡排序 - 扑克牌排序游戏

**零件清单**:
- 电脑（安装 Python 3.8+）
- 扑克牌 10 张（A-10，只用数字）
- 纸和笔（记录比较次数）

**玩法描述**:

1. 先用扑克牌手动体验冒泡排序：
   - 把 10 张牌打乱，面朝上放在桌上
   - 从左到右，相邻两张牌比较
   - 如果左边比右边大，就交换位置
   - 一轮下来，最大的牌会"冒泡"到最右边
   - 重复这个过程，直到所有牌排好序

2. 用 Python 实现冒泡排序：
```python
def bubble_sort(arr):
    n = len(arr)
    count = 0  # 记录比较次数
    
    for i in range(n - 1):
        for j in range(n - 1 - i):
            count += 1
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                print(f"交换: {arr[j+1]} 和 {arr[j]}")
        
        print(f"第{i+1}轮结果: {arr}")
    
    print(f"总共比较了 {count} 次")
    return arr

# 测试
scores = [85, 92, 78, 95, 88, 76, 100, 65, 90, 82]
print(f"原始成绩: {scores}")
sorted_scores = bubble_sort(scores)
print(f"排序后: {sorted_scores}")
```

3. 实验变体：
   - 用 5 张牌和 10 张牌分别排序，比较比较次数
   - 尝试已经排好序的数据，观察比较次数
   - 记录每轮的中间结果

**观察与思考**:
- 为什么叫"冒泡"排序？观察最大的数如何像气泡一样浮到右边
- 5 张牌需要比较多少次？10 张牌呢？有什么规律？
- 如果数据已经排好序，冒泡排序还需要比较吗？怎么优化？

### 实验2: 🔍 二分搜索 - 猜数字游戏

**零件清单**:
- 电脑
- 纸和笔
- 一个朋友或家长（一起玩）

**玩法描述**:

1. 先玩猜数字游戏：
   - 心里想一个 1-100 的数字
   - 对方每次猜一个数，你只能回答"大了"、"小了"或"对了"
   - 记录猜了多少次才猜对

2. 用 Python 实现二分搜索：
```python
def binary_search(arr, target):
    left = 0
    right = len(arr) - 1
    count = 0
    
    while left <= right:
        count += 1
        mid = (left + right) // 2
        print(f"第{count}次猜测: 位置{mid}, 值{arr[mid]}")
        
        if arr[mid] == target:
            print(f"找到了！{target} 在位置 {mid}")
            print(f"总共猜了 {count} 次")
            return mid
        elif arr[mid] < target:
            print("小了，往右找")
            left = mid + 1
        else:
            print("大了，往左找")
            right = mid - 1
    
    print(f"没找到 {target}")
    return -1

# 测试：在100个数中搜索
numbers = list(range(1, 101))  # [1, 2, 3, ..., 100]
print(f"在1-100中搜索数字 73")
result = binary_search(numbers, 73)
```

3. 对比实验：
   - 用线性搜索（从头到尾一个个找）找同一个数
   - 比较两种方法的查找次数
```python
def linear_search(arr, target):
    count = 0
    for i in range(len(arr)):
        count += 1
        if arr[i] == target:
            print(f"线性搜索: 找了 {count} 次")
            return i
    return -1

# 对比
linear_search(numbers, 73)  # 可能要找73次
binary_search(numbers, 73)  # 只需要7次左右
```

**观察与思考**:
- 为什么二分搜索必须先排序？
- 100 个数最多需要猜几次？1000 个数呢？
- 二分搜索和翻字典找单词有什么相似之处？

### 实验3: 🐢 递归 - 乌龟画图与数学之美

**零件清单**:
- 电脑（安装 Python 的 turtle 库，通常自带）
- 纸和笔（手动画递归图形）

**玩法描述**:

1. 先理解递归的概念：
   - 递归就像俄罗斯套娃，大娃娃里套小娃娃
   - 每个娃娃都长一样，只是大小不同
   - 递归必须有"出口"，不然会无限套下去

2. 用递归计算阶乘：
```python
def factorial(n):
    # 递归出口：0! = 1
    if n == 0:
        return 1
    # 递归公式：n! = n × (n-1)!
    return n * factorial(n - 1)

# 测试
print(f"5! = {factorial(5)}")  # 120
print(f"10! = {factorial(10)}")  # 3628800

# 画出递归调用过程
def factorial_with_trace(n, depth=0):
    indent = "  " * depth
    print(f"{indent}计算 {n}!")
    
    if n == 0:
        print(f"{indent}返回 1 (递归出口)")
        return 1
    
    result = n * factorial_with_trace(n - 1, depth + 1)
    print(f"{indent}返回 {n} × {result // n} = {result}")
    return result

factorial_with_trace(4)
```

3. 用递归画分形树：
```python
import turtle

def draw_tree(branch_length, t):
    if branch_length < 10:  # 递归出口：树枝太短就不画了
        return
    
    t.forward(branch_length)
    t.left(30)
    draw_tree(branch_length * 0.7, t)  # 画左子树
    t.right(60)
    draw_tree(branch_length * 0.7, t)  # 画右子树
    t.left(30)
    t.backward(branch_length)

# 设置画布
screen = turtle.Screen()
screen.bgcolor("white")
t = turtle.Turtle()
t.color("green")
t.left(90)
t.speed(0)

draw_tree(100, t)
screen.exitonclick()
```

4. 手动练习：
   - 在纸上画出 factorial(4) 的调用过程
   - 用箭头表示递归调用和返回
   - 标出每个函数调用的参数和返回值

**观察与思考**:
- 递归和循环有什么区别？什么时候用递归更好？
- 如果递归没有出口会怎样？（试试 factorial(-1)）
- 生活中有哪些递归的例子？（如：文件夹套文件夹）

### 挑战1: 选择排序
实现选择排序算法。每一轮找到最小的数，放到前面。比较它和冒泡排序的效率。

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

## 📝 家长小贴士

> 💡 **关键点**: 算法学习重在理解思想，不是背代码。让孩子先用扑克牌、积木等实物体验排序过程，再写代码。理解了"为什么要这样做"，代码自然就写出来了。

> ⚠️ **常见误区**: 不要一开始就教高级算法（如快速排序、动态规划）。10-12 岁的孩子先把冒泡排序、二分搜索、递归这三个基础打牢，比囫囵吞枣学十个算法更有效。竞赛考的是思维深度，不是算法数量。

> 🎯 **练习建议**: 每周做 2-3 道算法题，从洛谷或力扣的入门题开始。重在坚持，不在数量。做完一道题，让孩子讲一遍思路，比闷头做十道题更有效。
