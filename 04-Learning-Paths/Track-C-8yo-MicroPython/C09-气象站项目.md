## 🧠 核心原理简述（给家长）

**气象站**是物联网的经典应用。它集成了多种传感器，实时监测环境数据。

项目架构：
1. **数据采集**: 温湿度、气压、光照等传感器
2. **数据处理**: 滤波、校准、单位转换
3. **数据展示**: OLED屏幕本地显示
4. **数据上传**: WiFi上传到云端平台
5. **数据分析**: 历史数据、趋势图表

孩子将学习：
- **系统设计**: 如何设计一个完整系统
- **多传感器融合**: 综合多个传感器数据
- **用户界面**: 设计友好的显示界面

## 🔬 趣味实验清单

### 实验1: 🌡️ 温湿度监测

**零件清单**:
- ESP32开发板一块
- DHT11温湿度传感器
- OLED显示屏（0.96寸，I2C接口）
- 面包板
- 杜邦线若干
- 电脑

**玩法描述**:
1. 连接电路：
   - DHT11数据引脚 → GPIO4
   - OLED SDA → GPIO21
   - OLED SCL → GPIO22
2. 编写温湿度监测程序：
```python
import machine
import dht
import time
from ssd1306 import SSD1306_I2C

# 传感器和显示初始化
sensor = dht.DHT11(machine.Pin(4))
i2c = machine.I2C(scl=machine.Pin(22), sda=machine.Pin(21))
oled = SSD1306_I2C(128, 64, i2c)

while True:
    sensor.measure()
    temp = sensor.temperature()
    humi = sensor.humidity()
    
    # 清屏
    oled.fill(0)
    
    # 显示数据
    oled.text("Weather Station", 0, 0)
    oled.text("Temp: {}C".format(temp), 0, 20)
    oled.text("Humi: {}%".format(humi), 0, 40)
    
    # 更新显示
    oled.show()
    
    time.sleep(2)
```
3. 下载程序，观察OLED显示
4. 用手握住传感器，观察数据变化

**观察与思考**:
- 温度和湿度有什么关系？
- 数据更新频率是多少合适？
- 如何让显示更美观？

### 实验2: 📺 OLED显示

**零件清单**:
- 同实验1

**玩法描述**:
1. 设计更丰富的显示界面：
```python
import machine
import dht
import time
from ssd1306 import SSD1306_I2C

sensor = dht.DHT11(machine.Pin(4))
i2c = machine.I2C(scl=machine.Pin(22), sda=machine.Pin(21))
oled = SSD1306_I2C(128, 64, i2c)

# 历史数据
temp_history = []
humi_history = []

while True:
    sensor.measure()
    temp = sensor.temperature()
    humi = sensor.humidity()
    
    # 保存历史数据
    temp_history.append(temp)
    humi_history.append(humi)
    if len(temp_history) > 10:
        temp_history.pop(0)
        humi_history.pop(0)
    
    # 清屏
    oled.fill(0)
    
    # 标题
    oled.text("Weather Station", 0, 0)
    
    # 当前数据
    oled.text("T:{}".format(temp), 0, 16)
    oled.text("H:{}".format(humi), 64, 16)
    
    # 温度条形图
    oled.rect(0, 32, 128, 8, 1)
    bar_width = int(temp * 128 / 50)  # 假设最高50度
    oled.fill_rect(0, 32, bar_width, 8, 1)
    
    # 趋势箭头
    if len(temp_history) > 1:
        if temp > temp_history[-2]:
            oled.text("^", 100, 16)  # 上升
        elif temp < temp_history[-2]:
            oled.text("v", 100, 16)  # 下降
        else:
            oled.text("=", 100, 16)  # 持平
    
    oled.show()
    time.sleep(2)
```
2. 观察条形图和趋势箭头
3. 讨论如何设计更好的界面

**观察与思考**:
- 条形图和数字，哪个更直观？
- 趋势箭头有什么用？
- 如何显示历史数据曲线？

### 实验3: 📤 数据上传

**零件清单**:
- ESP32开发板一块
- DHT11传感器
- OLED显示屏
- 电脑
- ThingSpeak账号（免费注册）

**玩法描述**:
1. 注册ThingSpeak账号，创建通道
2. 编写数据上传程序：
```python
import network
import socket
import dht
import machine
import time
import urequests

# WiFi配置
SSID = "你的WiFi名称"
PASSWORD = "你的WiFi密码"

# ThingSpeak配置
API_KEY = "你的Write API Key"

# 连接WiFi
wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(SSID, PASSWORD)

while not wlan.isconnected():
    time.sleep(1)

print("已连接:", wlan.ifconfig()[0])

# 传感器
sensor = dht.DHT11(machine.Pin(4))

while True:
    try:
        sensor.measure()
        temp = sensor.temperature()
        humi = sensor.humidity()
        
        # 上传到ThingSpeak
        url = "https://api.thingspeak.com/update?api_key={}&field1={}&field2={}".format(API_KEY, temp, humi)
        response = urequests.get(url)
        print("上传成功:", response.text)
        response.close()
        
    except Exception as e:
        print("错误:", e)
    
    time.sleep(30)  # ThingSpeak免费版每15秒只能更新一次
```
3. 下载程序，观察数据上传
4. 登录ThingSpeak查看数据图表

**观察与思考**:
- ThingSpeak上的数据图表是什么样的？
- 数据更新频率有什么限制？
- 还有哪些云平台可以使用？
